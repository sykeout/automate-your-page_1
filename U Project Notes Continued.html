Web Basics

How it works

The internet is a truck load of interconnected computers that can communicate with one another. When an internet user requests a web page by typing in a url like https://www.udacity.com their computer sends the page request to a server (another computer) which returns the html document to be read by the users browser of choice.

html explained

html is an acronym for hypertext-markup-language. This html code/text is used to control what users see on the internet. Most of it at least.

Tags and Elements

In an effort to control the structure of a web page, when we code with html we need to explain to the browser what each element of the page actually is, so it knows what to do with them. We do this by tagging each element.

Puters are Stupid

Computers are literal, and only understand code as it is supposed to me. This means that tiny mistakes, that are very hard to see for a human, can make big differences to what is presented in the browser.

Inline vs. Block

All html elements are either inline, which is to say they follow on from one another, or they are block elements. block elements are placed inside a box, which has many additional properties that can be manipulated to control everything from position on the page to borders.

Structured Documents

Developer Tools

Google provides tools that enable developers to see and play with the html and css code of existing web pages. This can be very helpful to help understand how subtle changes in the code make a difference to how the page is shown in the browser.

Tree Structure

html elements can be placed inside other html elements, many times over. This means that in the code they are written like a family tree, where one parent has two children within it, which in turn have 3, 4 or any number of children, and so on.

Indentation

When you write html code, or look at it, the lines of code are indented depending on how far nested that line of code is within another line of code.

Text Editors

There are many software applications that are built to make writing code as easy and efficient as possible, called text editors. I'm using sublime text right now.

Repeatition is the enemy of coding, and should be avoid at all costs. Efficiencies with coding itself, as well as structure should be used instead. Because so many elements within a webpage, or entire website are very similar, web developers would have to write the same style code over and over. One prime example of reducing repreatition os the use of css to control the style of a html web page, instead of coding it within html.

Programming Languages

Programming languages were created to overcome the limitations of natural languages, which are unsuitable for the literal intepretation computers are currently capable of. The two biggest downfalls of natural languages, are:

  • Ambiguity
  • Verbosity

Python for Arithmetic

Python requires perfect grammar for arithmetic otherwise a syntax error will occur. Don't panic if this happens. Simply debug, and refer back to the basic grammar rules:

  • Expression -> Expression Operator Expression
  • Expression -> Number
  • Operator -> + * / - etc
  • Number -> 0,1,2,3,4......
  • Expression -> (Expression)

Py calculates in integers by default, so to force it to poresent answers after the decimal you need to make one of the values have a decimal, like using 5.0 instead of simply 5.

What is a variable?

A variable is a way of storing a string of information under a name.

In py the = sign doesn't mean equals, it means assignment and is used to assign a name to a string.

We create variables via an 'assignment statement', like this: my_variable = "let me out, I keep getting stuck in here". This means from this point forward (unless changed later) 'my_variable' means "let me out, I keep getting stuck in here".

And strings?

A string is a string of characaters, basically anything we can type into a keyboard.

Strings must open and close with either "" OR '', but not a combination of those.

We can index strings, meaning that we can access any specific part of the string and do something with it. A simple example os printing a certain character from a specific location within the string. An example might be

my_name = "christiansykes"

print my_name[4] would print out the character at position 4 of my_name, i.e the letter 's'.

print my_name[1+1] would print out the character at position 2, ie: r

print my_name[-1] would print out the last charcter in the string

Selecting Sub-sequences of a String

Taking indexing one step further, we can select a range of characters from within the string.

print my_name[0:3] would print: chr

print my_name[3:] woudl print: istiansykes

Find within strings

We can also search for specific charcters within a string, using an in-build method called 'find'.

We do this with: variable_name.find('value')

print my_name.find('stia') would return: 4

We can further controld find, by specifying where in the string find is to start, we do this like:

print my_name.find('ia', 3)

Functions // Proceedures

We can create and save code proceedures that complete a task upon 1 or more inputs. They simply take inputs and do something with them, before returning a result.

First we define the proceedure, ike this:

def pro(input_a, input_b):

result = input_a + input_b

return result

To use, or "call", the proceedure, we pass in some inputs as per the defintition, like this:

print pro(4,5)

This would return 9

If the proceedure doesn't include a return statement, it could be said to do nothing, and therefore is of little use.

Proceedures are excellent for the avoidance of repetition, as they can be called as often as needed to complete the task they are defined to do.

If Statements

We might sometimes want to have a function run only IF inputs meet certain criteria, essentially adding simple decision making to the code. If statements allow us to do just that.

A simple if statement might look like this:

if A > b:

return a

This would check to see wich value was higher, and if a was, it would return that value.

We can add in another line of code so that if a is not greater than b, the code returns b.

This could be done ina few ways, here are two examples:

  1. return b - long as this was indented to the same degree than the "if" statement was
  2. else: return b

Loop a While

Sometimes we might want code to keep on doing something, untill a certain criteria is met. We can do this with loops.

One kind of loop, is called the 'While Loop'.

This might look like this:

while a != b:

return a

a = a + 1

We'd set the variable a to a certain number first, and then the loop would keep printing the value of a, adding 1 to it, and then checking to see if it is not equal to b again. And so on.

Lists

Similar to strings, but with interesting addtional functionality and capabilities.

Can include aythign at all, including other lists.

Lists support mutuation:- You can make a change to the variable, rather than simply changing the assignement of the variable name/definition to a new variable.

Aliasing is possible:- Two definitions refer to the same variable. When the value is changed, it changes for both aliases.

List Operations:

  • Apend:- you can add values to new positions at the end of the list.
  • Plus:- You can join two strings together, like contatenation of two halves.
  • len // Length returns the number of elements in the list

Looping through a list:- We are introduced to "For loops" that are motivated to run throguh all elements ina list
